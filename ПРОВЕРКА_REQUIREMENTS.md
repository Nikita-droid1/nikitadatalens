# Проверка requirements.txt

## Проблема

Workflow все равно не находит `requirements.txt`, даже после добавления.

## Возможные причины

1. **Файл не в корне репозитория**
   - Должен быть: `requirements.txt` (в корне)
   - НЕ должно быть: `папка/requirements.txt`

2. **Файл не закоммичен**
   - Файл создан, но не закоммичен/не запушен

3. **Неправильное имя файла**
   - Должно быть: `requirements.txt`
   - НЕ должно быть: `REQUIREMENTS.txt`, `requirements.TXT` и т.д.

## Как проверить в GitHub

1. Открой свой репозиторий на GitHub
2. Убедись, что файл `requirements.txt` виден в **корне** репозитория (не в подпапке)
3. Открой файл и проверь содержимое

## Решение: Добавить файл заново

### Вариант 1: Через веб-интерфейс (гарантированно работает)

1. Открой репозиторий на GitHub
2. Нажми **"Add file"** → **"Create new file"**
3. В поле имени файла введи ТОЧНО: `requirements.txt` (в корне, не в папке!)
4. Скопируй содержимое из файла `REQUIREMENTS_ДЛЯ_КОПИРОВАНИЯ.txt`
5. Вставь в редактор
6. Нажми **"Commit new file"**
7. Проверь, что файл появился в корне репозитория

### Вариант 2: Через терминал (если есть доступ)

```bash
cd /путь/к/репозиторию
# Убедись, что находишься в корне репозитория
pwd

# Создай файл requirements.txt
cat > requirements.txt << 'EOF'
# Python зависимости для проекта дата-аналитики Room Broom

# HTTP запросы
requests>=2.31.0

# Работа с БД PostgreSQL (Neon)
psycopg2-binary>=2.9.9

# Работа с Google Sheets API
gspread>=5.12.0
google-auth>=2.23.0
google-auth-oauthlib>=1.1.0
google-auth-httplib2>=0.1.1

# Работа с данными
pandas>=2.1.0

# Переменные окружения
python-dotenv>=1.0.0
EOF

# Закоммить
git add requirements.txt
git commit -m "Add requirements.txt"
git push
```

## Проверка после добавления

1. Открой репозиторий на GitHub
2. Убедись, что файл `requirements.txt` виден в **корне** (на одном уровне с README.md, etl.py и т.д.)
3. Перезапусти workflow
4. Ошибка должна исчезнуть

## Если все равно не работает

Проверь структуру репозитория в GitHub:
- Должны быть файлы: `requirements.txt`, `etl.py`, `README.md` в корне
- Должна быть папка: `.github/workflows/` с файлом workflow

Если структура другая — возможно, код загружен в подпапку. Нужно будет скорректировать workflow или переместить файлы.
