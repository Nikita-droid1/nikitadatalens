# Быстрое исправление: добавить requirements.txt в GitHub

## Проблема

Workflow запустился, но падает с ошибкой: `requirements.txt not found`

## Решение (2 минуты)

### Шаг 1: Добавить файл в GitHub

1. Открой свой репозиторий на GitHub
2. Нажми **"Add file"** → **"Create new file"**
3. В поле имени файла введи: `requirements.txt`
4. Открой файл `REQUIREMENTS_ДЛЯ_КОПИРОВАНИЯ.txt` в папке `1_дата-аналитика`
5. Скопируй **весь** текст из этого файла
6. Вставь в редактор GitHub
7. Нажми **"Commit new file"**

### Шаг 2: Перезапустить workflow

1. Перейди во вкладку **Actions**
2. Открой последний запуск workflow
3. Нажми **"Re-run jobs"** → **"Re-run failed jobs"**

Или просто запусти workflow заново: **Actions** → **"ETL (ручной запуск)"** → **"Run workflow"**

## Что дальше

После исправления workflow пройдет дальше, но может упасть на следующем шаге (если нет секретов) — это нормально. 

**Следующий шаг:** Добавь секреты в GitHub (ШАГ 1 из чеклиста).
