# Инструкция по отслеживанию изменений для GitHub

## Как это работает

1. **Основная папка:** `/Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика`
   - Здесь ты работаешь и вносишь изменения
   - Здесь есть файл `.env` с секретами (не попадает в GitHub)

2. **Папка для GitHub:** `/Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика_github`
   - Чистая копия проекта без секретов
   - Отсюда загружается код в GitHub репозиторий

## Когда я буду говорить об изменениях

Я буду отслеживать изменения в основной папке и сообщать тебе:
- ✅ Какие новые файлы нужно добавить в GitHub
- ✅ Какие файлы изменились и нужно обновить
- ✅ Что именно делать (команды для синхронизации и коммита)

## Как обновить папку для GitHub

### Вариант 1: Через скрипт (проще)

```bash
cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика
./sync_to_github.sh
```

### Вариант 2: Вручную

Просто скажи мне: **"Обнови папку для GitHub"** — я выполню синхронизацию.

## Как загрузить изменения в GitHub

После синхронизации:

```bash
cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика_github
git add .
git commit -m "Описание изменений"
git push
```

Или скажи мне: **"Закоммить изменения в GitHub"** — я помогу с командами.

## Что отслеживается

- Новые файлы (Python, SQL, документация, конфиги)
- Изменения в существующих файлах
- Удаление файлов
- Изменения в workflow файлах

## Что НЕ попадает в GitHub

- `.env` (секреты)
- `.git` (локальная история Git)
- `__pycache__` (кэш Python)
- `.DS_Store` (системные файлы Mac)
- `.venv`, `venv` (виртуальные окружения)

---

**Важно:** Всегда проверяй, что `.env` не попал в папку для GitHub перед коммитом!
