# Простая инструкция: Как настроить Git и GitHub в Cursor

## Главное

**Cursor уже умеет работать с Git/GitHub** — не нужны дополнительные плагины или MCP. Нужно только правильно настроить.

---

## Что происходит сейчас

Ты загрузил код в GitHub вручную (через веб-интерфейс или другую папку). Но **основная папка `1_дата-аналитика` не подключена к GitHub**, поэтому изменения не синхронизируются автоматически.

---

## Решение: Подключить папку к GitHub

### Вариант 1: Через Cursor (проще всего)

1. **Открой папку проекта в Cursor:**
   - File → Open Folder
   - Выбери: `/Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика`

2. **Открой панель Source Control:**
   - Нажми иконку ветки слева (или `Cmd+Shift+G`)
   - Если видишь список файлов — Git уже работает ✅
   - Если видишь "Initialize Repository" — нажми эту кнопку

3. **Подключи к GitHub:**
   - Нажми **"..."** (три точки) в панели Source Control
   - Выбери **"Remote"** → **"Add Remote"**
   - Введи имя: `origin`
   - Введи URL: `https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git`
   - Нажми **"Add"**

4. **Отправь код:**
   - Нажми **"..."** → **"Push"**
   - Выбери `origin` и ветку `main`

### Вариант 2: Через терминал в Cursor

Открой терминал в Cursor (Terminal → New Terminal) и выполни:

```bash
cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика

# Инициализируй Git (если еще не сделано)
git init

# Добавь все файлы
git add .

# Создай первый коммит
git commit -m "Initial commit: ETL pipeline"

# Подключи к GitHub (замени на свою ссылку)
git remote add origin https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

# Отправь код
git push -u origin main
```

---

## Как это будет работать дальше

После настройки:

1. **Ты меняешь код** в Cursor
2. **Cursor видит изменения** (в панели Source Control)
3. **Ты делаешь коммит** (кнопка Commit)
4. **Ты пушишь в GitHub** (кнопка Push)
5. **GitHub Actions запускается** (если настроен workflow)

**Автоматически:** Cursor видит изменения и показывает их в панели Source Control.

**Вручную:** Ты делаешь коммит и пуш (это нормально, так работает Git).

---

## Что делать при изменениях

Когда я что-то меняю в коде или ты добавляешь файлы:

1. **Открой панель Source Control** (`Cmd+Shift+G`)
2. **Увидишь список измененных файлов**
3. **Нажми "+"** рядом с файлами, чтобы добавить в коммит
4. **Введи сообщение** (например, "Добавлен новый модуль")
5. **Нажми "Commit"** (галочка)
6. **Нажми "..." → "Push"** → выбери `origin`

**Или через терминал:**
```bash
git add .
git commit -m "Описание изменений"
git push
```

## Если что-то не работает

**Проблема:** Не вижу панель Source Control
- **Решение:** View → Source Control (или `Cmd+Shift+G`)

**Проблема:** Не могу подключить репозиторий
- **Решение:** Проверь ссылку на GitHub репозиторий (должна быть вида `https://github.com/...`)

**Проблема:** Не могу запушить
- **Решение:** Возможно, нужна авторизация в GitHub (токен доступа)
