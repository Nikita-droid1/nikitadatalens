# Основы Git для проекта

## Что такое Git и зачем он нужен?

**Git** — это система контроля версий. Она позволяет:
- Сохранять историю изменений кода
- Откатываться к предыдущим версиям
- Работать в команде без конфликтов
- Хранить код в облаке (GitHub)

**GitHub** — это сайт, где хранится код в облаке.

---

## Основные понятия

### Репозиторий (repo)
Папка с проектом, которая отслеживается Git. Может быть:
- **Локальный** — на твоем компьютере
- **Удаленный** — на GitHub

### Коммит (commit)
Снимок состояния кода в определенный момент времени. Это как "сохранение" в игре, но для кода.

### Пуш (push)
Отправка коммитов из локального репозитория на GitHub.

---

## Как закоммитить код проекта

### Вариант 1: Через терминал (командная строка)

1. **Открой терминал** (Terminal на Mac)

2. **Перейди в папку проекта:**
   ```bash
   cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика
   ```

3. **Проверь, что Git установлен:**
   ```bash
   git --version
   ```
   Если команда не найдена — установи Git с [git-scm.com](https://git-scm.com/)

4. **Инициализируй Git в папке (если еще не сделано):**
   ```bash
   git init
   ```

5. **Добавь все файлы в "корзину" для коммита:**
   ```bash
   git add .
   ```
   Точка означает "все файлы в текущей папке"

6. **Создай коммит (сохранение):**
   ```bash
   git commit -m "Initial commit: ETL pipeline for Room Broom analytics"
   ```
   `-m` означает "message" — описание того, что изменилось

7. **Подключи удаленный репозиторий (если еще не подключен):**
   ```bash
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
   ```
   Замени на реальную ссылку своего репозитория на GitHub

8. **Отправь код на GitHub:**
   ```bash
   git push -u origin main
   ```
   Или `git push -u origin master` (зависит от названия ветки)

---

### Вариант 2: Через GitHub Desktop (проще для новичков)

1. **Скачай GitHub Desktop:** [desktop.github.com](https://desktop.github.com/)

2. **Установи и войди** в свой GitHub аккаунт

3. **Добавь репозиторий:**
   - File → Add Local Repository
   - Выбери папку проекта: `/Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика`

4. **Создай коммит:**
   - Внизу слева напиши описание: "Initial commit: ETL pipeline"
   - Нажми кнопку **"Commit to main"**

5. **Отправь на GitHub:**
   - Нажми кнопку **"Push origin"** или **"Publish repository"**

---

### Вариант 3: Через VS Code / Cursor (если используешь)

1. Открой папку проекта в Cursor/VS Code

2. Открой вкладку **Source Control** (иконка ветки слева)

3. Нажми кнопку **"Initialize Repository"** (если Git еще не инициализирован)

4. Нажми **"+"** рядом с файлами, чтобы добавить их в коммит

5. Введите сообщение коммита вверху: "Initial commit: ETL pipeline"

6. Нажми кнопку **"Commit"** (галочка)

7. Нажми **"..."** → **"Push"** → выбери репозиторий на GitHub

---

## Что происходит после коммита?

После того как ты закоммитил и запушил код:
1. Все файлы проекта попадают на GitHub
2. Файл `.github/workflows/daily_etl.yml` становится доступен GitHub Actions
3. Во вкладке **Actions** появляется workflow "Daily ETL"
4. Теперь можно запускать ETL через GitHub

---

## Важно!

- Файл `.env` **НЕ должен** попасть в репозиторий (он в `.gitignore`)
- Если случайно добавил `.env` — удали его из репозитория:
  ```bash
  git rm --cached .env
  git commit -m "Remove .env from repository"
  git push
  ```

---

## Проверка

После коммита и пуша:
1. Открой свой репозиторий на GitHub
2. Убедись, что все файлы видны
3. Перейди во вкладку **Actions**
4. Должен появиться workflow "Daily ETL"
