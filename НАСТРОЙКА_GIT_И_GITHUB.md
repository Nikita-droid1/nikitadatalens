# Настройка Git и GitHub в Cursor

## Как это работает

**Cursor уже имеет встроенную интеграцию с Git/GitHub** — не нужны дополнительные MCP или плагины.

**Что происходит:**
1. Cursor видит Git репозиторий в папке проекта
2. Может делать коммиты и пушить в GitHub
3. Может видеть изменения и историю

**Проблема:** Нужно правильно настроить Git и подключить удаленный репозиторий.

---

## Шаг 1: Проверка текущего состояния

Открой терминал в Cursor (Terminal → New Terminal) и выполни:

```bash
cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика
git status
```

**Что проверить:**
- Если видишь список файлов — Git инициализирован ✅
- Если ошибка "not a git repository" — нужно инициализировать Git

---

## Шаг 2: Инициализация Git (если еще не сделано)

```bash
cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика
git init
git add .
git commit -m "Initial commit: ETL pipeline"
```

---

## Шаг 3: Подключение к GitHub репозиторию

### Вариант А: Если репозиторий уже создан на GitHub

```bash
# Замени на свою ссылку из GitHub
git remote add origin https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

# Проверь подключение
git remote -v

# Отправь код на GitHub
git push -u origin main
```

### Вариант Б: Если репозитория еще нет

1. Создай репозиторий на GitHub (New repository)
2. НЕ ставь галочки на "Initialize with README"
3. Скопируй ссылку репозитория
4. Выполни команды из Варианта А

---

## Шаг 4: Использование Git в Cursor

### Способ 1: Через встроенный интерфейс (проще)

1. Открой панель **Source Control** (иконка ветки слева, или `Cmd+Shift+G`)
2. Там будут видны все изменения
3. Нажми **"+"** рядом с файлами, чтобы добавить в коммит
4. Введи сообщение коммита вверху
5. Нажми **"Commit"** (галочка)
6. Нажми **"..."** → **"Push"** → выбери репозиторий

### Способ 2: Через терминал

```bash
git add .
git commit -m "Описание изменений"
git push
```

---

## Шаг 5: Автоматическая синхронизация

**К сожалению, полностью автоматической синхронизации нет** — нужно вручную делать коммиты и пуш.

**Но можно упростить:**

1. **Используй панель Source Control в Cursor** — там видно все изменения
2. **Делай коммиты регулярно** — после каждого изменения
3. **Используй кнопку Push** — чтобы отправить на GitHub

---

## Что делать при изменениях

Когда ты что-то меняешь в коде:

1. **Cursor автоматически видит изменения** (в панели Source Control)
2. **Ты делаешь коммит** (через панель или терминал)
3. **Ты пушишь в GitHub** (через панель или `git push`)
4. **GitHub Actions автоматически запускается** (если настроен workflow)

---

## Проверка, что все работает

1. **Проверь Git:**
   ```bash
   git status
   git remote -v
   ```

2. **Проверь GitHub:**
   - Открой репозиторий на GitHub
   - Убедись, что все файлы видны
   - Проверь вкладку Actions

3. **Проверь Cursor:**
   - Открой панель Source Control (`Cmd+Shift+G`)
   - Должны быть видны изменения (если есть)

---

## Если что-то не работает

### Git не видит изменения?

```bash
git status
```

Если файлы не видны — возможно, они в `.gitignore`. Проверь файл `.gitignore`.

### Не могу подключить репозиторий?

Проверь ссылку — она должна быть вида:
```
https://github.com/логин/название-репо.git
```

### Не могу запушить?

Возможно, нужна авторизация. GitHub может попросить токен доступа.
